Model Input Data Information

About the Model Input Data

The model input data consists of the final processed datasets that are ready for model training. These datasets have been transformed into the specific format required by our deep learning model.

Training Data Structure
Dataset Name Description Format
X_train.npy Training features (24x24x24 grid, hourly data) NumPy Array
y_train.npy Training labels (24x24x1 grid, next hour demand) NumPy Array
X_val.npy Validation features NumPy Array
y_val.npy Validation labels NumPy Array
X_test.npy Test features NumPy Array
y_test.npy Test labels NumPy Array
Feature Engineering

The following features have been engineered for the model:

  • Temporal Features:
    • Hour of day (0-23)
    • Day of week (0-6)
    • Month (1-12)
    • Holiday indicators
  • Spatial Features:
    • Grid cell coordinates
    • Neighborhood information
    • POI (Points of Interest) density
  • Weather Features:
    • Temperature
    • Precipitation
    • Wind speed
    • Weather conditions
Data Splits
  • Training set: 70% of the data
  • Validation set: 15% of the data
  • Test set: 15% of the data
Note: These datasets are in the final format required by the deep learning model and are used for training, validation, and testing.
